# PCB/TCB 用户态多任务模拟器

## 一、项目概述

### 1.1 项目目标
基于 Linux 用户态标准 C 库，使用 `ucontext` 实现独立栈与上下文切换，搭配 `timerfd` 模拟硬件时钟中断，手动模拟 PCB/TCB 任务控制块、时间片轮转调度；结合 `fork` 区分进程与线程资源隔离模型，先吃透操作系统多任务软件模型，后续再拓展 ARM64 裸机底层实现。

### 1.2 核心价值
- **理论验证**：在用户态验证操作系统核心概念（PCB、上下文切换、调度算法）
- **学习桥梁**：从软件抽象过渡到硬件实现的关键一步
- **无依赖环境**：普通 Linux 环境即可运行，无需 QEMU/裸机开发环境

## 二、核心模块设计

### 2.1 任务控制块模块（PCB/TCB）

**功能描述**：定义任务控制块结构体，统一管理所有任务

**数据结构设计**：
```c
typedef struct PCB {
    int pid;                    // 进程/线程ID
    enum TaskState state;       // 任务状态：RUNNING/READY/BLOCKED
    int time_slice;             // 剩余时间片
    int priority;               // 优先级（预留）
    void *stack;                // 独立栈地址
    ucontext_t context;         // ucontext上下文
    void *resource;             // 资源标记（进程/线程区分）
    struct PCB *next;           // 链表节点
} PCB;
```

**任务管理方式**：链表/数组统一管理

### 2.2 任务创建模块

**功能描述**：创建新任务，分配资源并加入就绪队列

**实现步骤**：
1. `malloc` 分配私有栈（8KB~16KB）
2. 使用 `getcontext` 初始化上下文
3. 设置栈指针和入口函数
4. 使用 `makecontext` 设置上下文
5. 填充 PCB 信息
6. 加入任务就绪队列

### 2.3 时钟中断模拟模块

**功能描述**：使用 timerfd 模拟周期性时钟中断

**实现机制**：
1. 创建 timerfd 文件描述符
2. 设置周期性定时（如 10ms）
3. 创建独立线程监听定时器
4. 中断到达时设置全局调度标记

### 2.4 轮转调度器模块

**功能描述**：检测时钟中断，执行上下文切换

**调度流程**：
1. 检测全局调度标记
2. 遍历 PCB 列表筛选就绪态任务
3. 保存当前任务上下文
4. 切换到下一个就绪任务
5. 恢复新任务上下文

### 2.5 任务状态管理

**支持状态**：
- `RUNNING`：正在运行
- `READY`：等待调度
- `BLOCKED`：阻塞等待

**状态转换**：
- RUNNING → READY：时间片耗尽
- READY → RUNNING：被调度选中
- RUNNING → BLOCKED：延时阻塞
- BLOCKED → READY：延时到期

**延时阻塞逻辑**：支持简单的 sleep 机制

### 2.6 进程对比拓展模块

**功能描述**：对比线程与进程的资源隔离差异

**对比维度**：
| 特性 | 线程（ucontext） | 进程（fork） |
|------|-----------------|-------------|
| 地址空间 | 共享 | 独立 |
| 栈 | 独立（malloc） | 独立（内核分配） |
| PCB | 用户态模拟 | 内核维护 |
| 调度 | 用户态轮转 | 内核调度 |

## 三、环境依赖

- **操作系统**：Linux x86_64 / ARM64
- **编译器**：GCC（支持 C11）
- **标准库**：glibc（提供 ucontext、timerfd 接口）
- **无需**：QEMU、汇编、裸机开发环境

## 四、学习递进路线

### 阶段 1：用户态模拟器（当前）
- 完成本用户态模拟器
- 掌握 PCB 结构、上下文切换、时间片调度、线程/进程概念

### 阶段 2：ARM64 架构学习
- 系统学习 ARM64 通用寄存器、系统寄存器
- 学习栈布局、汇编上下文保存恢复
- 理解异常处理机制

### 阶段 3：QEMU 裸机开发
- 切换至 QEMU ARM64 裸机开发
- 移除 libc/ucontext，手写汇编
- 实现 MMU、GIC、硬件定时器
- 实现真正底层操作系统调度
