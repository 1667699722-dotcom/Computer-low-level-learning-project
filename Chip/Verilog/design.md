# 五级流水线CPU设计文档

## 一、总体架构

### 1.1 流水线阶段划分

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│   IF    │───▶│   ID    │───▶│   EX    │───▶│   MEM   │───▶│   WB    │
│ 取指阶段 │    │ 译码阶段 │    │ 执行阶段 │    │ 访存阶段 │    │ 写回阶段 │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     ▲              ▲              ▲              ▲              ▲
     │              │              │              │              │
  IF/ID          ID/EX          EX/MEM         MEM/WB         寄存器堆
  流水线寄存器    流水线寄存器    流水线寄存器    流水线寄存器
```

### 1.2 设计目标

- **教学研究**：清晰展示流水线原理
- **结构简化**：突出核心概念，省略复杂特性
- **易于理解**：模块化设计，代码可读性强

## 二、指令集架构 (ISA)

### 2.1 指令格式

采用简化的 MIPS 风格指令集：

#### R 型指令（寄存器-寄存器）
```
31      26 25    21 20    16 15    11 10     6 5      0
┌─────────┬─────────┬─────────┬─────────┬─────────┬─────────┐
│  opcode │   rs    │   rt    │   rd    │  shamt  │ funct   │
│  6 bits │  5 bits │  5 bits │  5 bits │  5 bits │  6 bits │
└─────────┴─────────┴─────────┴─────────┴─────────┴─────────┘
```

#### I 型指令（立即数）
```
31      26 25    21 20    16 15                            0
┌─────────┬─────────┬─────────┬─────────────────────────────┐
│  opcode │   rs    │   rt    │         immediate           │
│  6 bits │  5 bits │  5 bits │          16 bits            │
└─────────┴─────────┴─────────┴─────────────────────────────┘
```

#### J 型指令（跳转）
```
31      26 25                                              0
┌─────────┬─────────────────────────────────────────────────┐
│  opcode │                   address                       │
│  6 bits │                   26 bits                       │
└─────────┴─────────────────────────────────────────────────┘
```

### 2.2 指令列表

| 指令 | 类型 | 功能描述 |
|------|------|----------|
| add  | R    | rd = rs + rt |
| sub  | R    | rd = rs - rt |
| and  | R    | rd = rs &amp; rt |
| or   | R    | rd = rs \| rt |
| slt  | R    | rd = (rs &lt; rt) ? 1 : 0 |
| addi | I    | rt = rs + immediate |
| lw   | I    | rt = memory[rs + immediate] |
| sw   | I    | memory[rs + immediate] = rt |
| beq  | I    | if (rs == rt) PC = PC + 4 + (immediate &lt;&lt; 2) |
| j    | J    | PC = (PC[31:28] &lt;&lt; 28) \| (address &lt;&lt; 2) |

## 三、流水线阶段详解

### 3.1 IF 阶段（取指阶段）

**功能**：从指令存储器取指令，计算下一条指令地址

**模块**：
- PC 寄存器
- 指令存储器 (IMEM)
- 加法器（PC + 4）

**数据通路**：
```
PC ──┬──▶ IMEM ──▶ 指令
     │
     └──▶ +4 ──▶ 下一条PC
```

### 3.2 ID 阶段（译码阶段）

**功能**：译码指令，读寄存器堆，符号扩展立即数

**模块**：
- 指令译码器
- 寄存器堆 (RegFile)
- 符号扩展单元
- 分支目标计算

**数据通路**：
```
指令 ──┬──▶ 译码器 ──▶ 控制信号
       ├──▶ RegFile ──▶ rs_data, rt_data
       └──▶ 符号扩展 ──▶ imm_ext
```

### 3.3 EX 阶段（执行阶段）

**功能**：执行算术运算，计算地址，前推处理

**模块**：
- ALU
- 前推单元 (Forwarding Unit)
- ALU 输入选择器

**数据通路**：
```
rs_data ──┬──▶ 前推 ──┐
rt_data ──┼──▶ 前推 ──┼──▶ ALU ──▶ ALU_result
imm_ext ──┘          │
                     └──▶ 零标志
```

### 3.4 MEM 阶段（访存阶段）

**功能**：访问数据存储器

**模块**：
- 数据存储器 (DMEM)

**数据通路**：
```
ALU_result ──┬──▶ DMEM 地址
rt_data ─────┼──▶ DMEM 写数据
             └──▶ DMEM 读数据
```

### 3.5 WB 阶段（写回阶段）

**功能**：将结果写回寄存器堆

**模块**：
- 写回选择器

**数据通路**：
```
ALU_result ──┐
             ├──▶ 选择器 ──▶ RegFile 写数据
DMEM_data ───┘
```

## 五、数据冒险处理

### 5.1 冒险类型

| 类型 | 说明 | 示例 |
|------|------|------|
| RAW | 写后读 | add $1, $2, $3; sub $4, $1, $5 |
| WAR | 读后写（本设计不处理） | |
| WAW | 写后写（本设计不处理） | |

### 5.2 前推（Forwarding）

**前推路径**：
1. EX/MEM → EX 阶段
2. MEM/WB → EX 阶段

**前推条件**：
```
前推 A (rs)：
- EX/MEM.reg_write 且 EX/MEM.write_reg != 0 且 EX/MEM.write_reg == ID/EX.rs
- 或 MEM/WB.reg_write 且 MEM/WB.write_reg != 0 且 MEM/WB.write_reg == ID/EX.rs

前推 B (rt)：
- EX/MEM.reg_write 且 EX/MEM.write_reg != 0 且 EX/MEM.write_reg == ID/EX.rt
- 或 MEM/WB.reg_write 且 MEM/WB.write_reg != 0 且 MEM/WB.write_reg == ID/EX.rt
```

### 5.3 气泡（Stall）

**Load-Use 冒险**：
```
lw  $1, 0($2)    # EX 阶段计算地址
sub $4, $1, $5   # ID 阶段需要 $1，此时 $1 还没准备好
```
需要插入一个气泡。

## 六、控制冒险处理

### 6.1 分支处理

**分支预测**：静态不预测（假设分支不发生）

**分支延迟**：在 ID 阶段计算分支目标和条件

**流水线冲刷**：分支发生时，冲刷 IF/ID 寄存器

## 十、学习要点

### 10.1 流水线优势
- 指令吞吐量提升
- 时钟周期缩短
- 硬件资源利用率提高

### 10.2 流水线挑战
- 数据冒险
- 控制冒险
- 结构冒险（本设计通过资源复制避免）

### 10.3 关键技术
- 流水线寄存器
- 数据前推
- 气泡插入
- 分支预测
